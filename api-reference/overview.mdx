## Base URL

```
https://makbool.ru/api/v1
```

## Аутентификация

Большинство эндпоинтов требуют JWT-токен пользователя. Исключение — `POST /errors`, который принимает API Key проекта в теле запроса.

### Получить токен

```http
POST /auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "your_password"
}
```

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer"
}
```

### Использовать токен

```http
GET /projects
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

## Формат ответов

Все ответы в формате JSON. Список объектов возвращается в обёртке:

```json
{
  "items": [...],
  "total": 42,
  "page": 1,
  "size": 20
}
```

## Коды ошибок

| Код | Значение |
|-----|---------|
| `400` | Неверный запрос — проверьте тело запроса |
| `401` | Не авторизован — передайте токен или API Key |
| `403` | Нет доступа |
| `404` | Ресурс не найден |
| `429` | Превышен лимит ошибок плана |
| `500` | Ошибка сервера |
