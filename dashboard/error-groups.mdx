## Список ошибок

На странице проекта отображается таблица групп ошибок. Каждая строка — это уникальная ошибка, сгруппированная по [fingerprint](/getting-started/concepts#fingerprinting).

### Что показывает таблица

| Колонка | Описание |
|---------|---------|
| **Ошибка** | Сообщение об ошибке и файл, где она произошла |
| **Окружение** | `production`, `staging` или другое |
| **Кол-во** | Сколько раз ошибка случилась всего |
| **Первый раз** | Когда ошибка появилась впервые |
| **Последний раз** | Когда произошла последний раз |
| **Статус** | `active` или `resolved` |

### Фильтры

Список можно фильтровать по:
- **Окружению** — `production`, `staging`, `development`
- **Статусу** — активные, решённые
- **Поиску** — по тексту сообщения

## Детали ошибки

Нажмите на строку, чтобы открыть страницу группы ошибок.

### Вкладка «Обзор»

- Сообщение и тип ошибки
- AI-объяснение (кнопка «Объяснить»)
- Статистика: количество, первый и последний случай
- Тренд за 24 часа

### Вкладка «Стектрейс»

Полный стектрейс с подсветкой синтаксиса. Если загружены [source maps](/dashboard/projects#source-maps), минифицированный стектрейс разворачивается в читаемый код с реальными именами функций и номерами строк.

### Вкладка «События»

Список отдельных событий внутри группы — каждое вхождение ошибки с:
- Временной меткой
- URL страницы, где произошла ошибка
- User agent браузера
- Версией релиза (`release`)
- Breadcrumbs — историей действий перед ошибкой

### Вкладка «Похожие ошибки»

Другие группы ошибок, семантически похожие на текущую. Помогает найти связанные проблемы.

## Управление статусом

### Отметить как решённую

Нажмите **«Решено»** на странице группы ошибок. Ошибка переходит в статус `resolved`.

Если та же ошибка появится снова — она автоматически вернётся в `active` и вы получите уведомление.

### Массовое управление

В списке ошибок выберите несколько групп и используйте кнопки в верхней панели для массовой смены статуса.

## Breadcrumbs

На вкладке «События» каждое событие содержит хронологическую ленту breadcrumbs — что происходило перед ошибкой:

```
12:34:01  navigation  Переход на /checkout
12:34:02  fetch       POST /api/orders → 200 (234ms)
12:34:03  fetch       POST /api/payment → 500 (1203ms)
12:34:04  error       TypeError: Cannot read properties of null
```

Это сильно ускоряет поиск причины ошибки — вы видите весь путь пользователя.
